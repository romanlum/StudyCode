\section{CaaS – Campina as a Service}
\subsection{Lösungsidee}
Bei dieser Aufgabe muss das Datenbankschema erstellt und dann pro Tabelle ein \textit{Dao} implementiert werden. \newline
Weiters wird dann der aktuelle BusinessLayer per RMI verwendet. Die neue Implementierung kann die \textit{Dao`s} verwenden um die DB Operation durchzuführen.

\subsubsection{Datenbank}
Folgendes Schema wird für die Datenbank verwendet. \newline
\includegraphics[width=200px]{schema.png}

Das Datenbankschema liegt im Projektordner unter \textbf{dbscripts/create-tables-mysql.sql} \newline\\
\textbf{Achtung:} Das Programm benötigt den MYSQL Treiber.
\subsubsection{Dao`s}
Für jede Tabelle des Schemas wird ein Dao Interface erstellt, wobei dies von einem generischen Basisinterface abgeleitet werden kann.  \newline
Die JDBC Implementierung kann generisch über \textit{Reflection} umgesetzt werden. Auch hier kann wieder eine generische Basisklasse verwendet werden.

\subsubsection{BusinessLayer - Dao Implementierung}
Der \textbf{Business-Layer} verwendet die \textbf{Dao`s} um die benötigten Operationen auszuführen. \newline

\subsubsection{RMI}
Die RMI Implementierung wurde der Einfachheit halber direkt in das Projekt integriert, sollte normalerweise aber in ein eigenes Projekt ausgelagert werden.\newline
Da in Übung 06 bereits ein Business-Layer verwendet wurde, kann dieser nun einfach per RMI Interface bereitgestellt werden.\newline
\newline
Der RMI Server Layer kann über die Klasse \textbf{at.lumetsnet.caas.rmi.RemoteServiceMain} gestartet werden.\newline
